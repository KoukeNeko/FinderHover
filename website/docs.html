初次設定

安裝 FinderHover 後，首次啟動需要完成以下設定：

  1. 從 Launchpad 或應用程式資料夾開啟 FinderHover
  2. App 會自動出現在選單列（Menu Bar）
  3. 系統會提示你授予輔助使用權限
  4. 完成授權後，將滑鼠移到 Finder 中的任何檔案上即可看到預覽
提示：你可以在設定中啟用「登入時啟動」，讓 FinderHover 在每次開機時自動執行。

授予權限

FinderHover 需要輔助使用權限才能偵測你目前指向的檔案。這是 macOS 的安全機制，確保只有經過授權的 App 才能存取這些資訊。

手動授予權限

  1. 開啟「系統設定」(System Settings)
  2. 前往「隱私與安全性」>「輔助使用」
  3. 找到 FinderHover 並開啟開關
  4. 如果看不到 FinderHover，點擊「+」按鈕手動加入
注意：授予權限後可能需要重新啟動 FinderHover 才能生效。

懸停預覽

FinderHover 的核心功能是在你將滑鼠停留在 Finder 中的檔案時，自動顯示該檔案的詳細資訊。

運作方式

  • 將滑鼠移到 Finder 視窗中的任何檔案或資料夾
  • 等待設定的延遲時間（預設 0.5 秒）
  • 預覽視窗會自動出現在滑鼠旁邊
  • 移動滑鼠到其他檔案會更新預覽內容
  • 移出 Finder 視窗或移到空白區域會隱藏預覽

智慧偵測

FinderHover 會自動偵測以下情況並暫時隱藏預覽：

  • 正在重新命名檔案
  • 正在拖曳檔案
  • 正在使用右鍵選單

偏好設定

點擊選單列圖示，然後選擇「偏好設定...」或按 ⌘ + , 開啟設定視窗。

一般設定

登入時啟動 開機時自動啟動 FinderHover
懸停延遲 滑鼠停留多久後顯示預覽（0.1 - 2.0 秒）
視窗大小 調整預覽視窗的尺寸
透明度 調整預覽視窗的透明度（70% - 100%）
字體縮放 調整預覽視窗中的文字大小

介面風格

FinderHover 提供兩種預覽視窗風格：

macOS 風格

豐富的視覺呈現，包含縮圖預覽、圖示和完整的中繼資料。適合需要詳細檔案資訊的使用者。

Windows 風格

簡約的工具提示樣式，只顯示基本資訊。適合偏好簡潔介面的使用者。

中繼資料欄位

你可以在設定中自訂要顯示哪些中繼資料欄位。以下是各類檔案支援的欄位：

影像檔案

  • 相機型號、鏡頭資訊、焦距
  • 光圈、快門速度、ISO
  • 拍攝日期、色彩空間、GPS 座標
  • IPTC/XMP（作者、版權、關鍵字、評分）
  • 色彩描述檔、位元深度、HDR 增益圖偵測

影片檔案

  • 影片時長、解析度、影片編碼
  • 幀率、位元率、影音軌道數量
  • HDR 格式（Dolby Vision、HDR10、HLG）
  • 章節數、字幕軌道、容器格式

音訊檔案

  • 曲目名稱、演出者、專輯
  • 專輯演出者、類型、年份
  • 時長、位元率、取樣率、聲道數

文件檔案

  • 文件標題、作者、主旨、關鍵字
  • 建立/修改日期、頁數、字數
  • 工作表數（Excel）、投影片數（PowerPoint）
  • PDF 版本、加密狀態、頁面尺寸

程式碼檔案

  • 程式語言自動偵測（38 種語言）
  • 總行數、程式碼行數
  • 註解行數、空白行數、檔案編碼

設定檔與資料格式

  • 格式類型、語法有效性
  • 鍵值數量、最大巢狀深度
  • 是否包含註解、檔案編碼

Markdown 檔案

  • Frontmatter 內容（YAML/TOML/JSON）
  • 文件標題（H1 或 Frontmatter）
  • 字數、標題數量、連結數量、圖片數量

網頁檔案

  • 頁面標題、描述、關鍵字
  • 作者、語言、字元編碼
  • Open Graph（og:title, og:description, og:image）
  • Twitter Card 類型

壓縮檔案

  • 壓縮格式類型
  • 檔案數量、解壓縮後大小
  • 壓縮比例、是否加密

電子書檔案

  • 書名、作者、出版商
  • 出版日期、ISBN、語言

字型檔案

  • 字型全名、字型家族
  • 字型風格（Regular、Bold 等）
  • 版本、設計師、版權資訊、字符數量

向量圖形與設計檔案

  • 格式類型、尺寸、ViewBox
  • 色彩模式（RGB、CMYK）
  • 圖層數量（PSD）、位元深度、解析度

字幕檔案

  • 字幕格式、文字編碼
  • 字幕條目數量、總時長
  • 語言代碼、是否包含格式化樣式

磁碟映像檔

  • 映像格式（UDIF、ISO 9660 等）
  • 總大小、壓縮後大小、壓縮比例
  • 加密狀態、分割區架構、檔案系統

資料庫檔案

  • 資料表數量、索引數量
  • 觸發器數量、檢視數量
  • Schema 版本、頁面大小、資料庫編碼

應用程式套件

  • Bundle ID、版本號、Build 號
  • 最低 macOS 版本需求
  • 應用程式類別、版權
  • 程式碼簽章狀態、Entitlements
  • 架構（arm64、x86_64、Universal）

Git 儲存庫

  • 目前分支、分支數量
  • Commit 數量、標籤數量
  • 最後 Commit 日期與訊息
  • 遠端 URL、未提交的變更狀態

系統中繼資料

  • Finder 標籤（顏色與自訂標籤）
  • 下載來源（來自哪個網站或應用程式）
  • 隔離資訊（Gatekeeper 狀態）
  • 連結資訊（符號連結、硬連結）
  • 使用統計（開啟次數、最後使用日期）
  • iCloud 狀態、Finder 註解、UTI 類型識別碼

3D 模型檔案

  • 格式類型（USDZ、OBJ、GLTF、FBX 等）
  • 頂點數、面數、網格數、材質數
  • 動畫數、骨架偵測、邊界框尺寸

Xcode 專案

  • 專案名稱、Target 數、建置配置數
  • Swift 版本、部署目標
  • 組織名稱、單元測試 / UI 測試偵測

檔案系統進階

  • 已分配磁碟空間（實際使用的區塊）
  • 屬性修改日期、Resource Fork 大小
  • 磁碟區資訊（名稱、格式、可用空間）
  • Spotlight 索引狀態
  • 雲端儲存偵測（iCloud、Dropbox、Google Drive、OneDrive）

疑難排解

預覽視窗沒有出現

  1. 確認 FinderHover 正在執行（選單列應該有圖示）
  2. 確認已授予輔助使用權限
  3. 嘗試重新啟動 FinderHover
  4. 確認滑鼠停留時間超過設定的延遲時間

權限授予後仍無法使用

  1. 完全退出 FinderHover
  2. 在系統設定中移除 FinderHover 的權限
  3. 重新開啟 FinderHover
  4. 重新授予權限

預覽視窗位置不正確

預覽視窗會自動調整位置以避免超出螢幕邊界。如果你使用多螢幕配置，可能會在某些邊界情況下出現位置問題。請回報此問題讓我們改進。
